## エピック1: ユーザー認証・初期設定

### ストーリー 1.1: 初回ログイン 🔵

**信頼性**: 🔵 *ヒアリングQ4: ソーシャルログイン選択より*

**私は** 新規ユーザー **として**
**Googleアカウントで素早くサインアップしたい**
**そうすることで** パスワードを管理せずにアプリを使い始められる

**関連要件**: REQ-101, REQ-102

**詳細シナリオ**:
1. トップページで「Googleでログイン」ボタンをクリック
2. Googleの認証画面が表示され、アカウントを選択
3. 認証後、初回オンボーディング画面（Wanna Be設定）へ遷移

**前提条件**:
- ユーザーがGoogleまたはAppleアカウントを持っている

**優先度**: Must Have

---

### ストーリー 1.2: Wanna Be 初期設定 🔵

**信頼性**: 🔵 *ヒアリングQ6より*

**私は** 初めてアプリを使うユーザー **として**
**「なりたい自分」を自由な言葉で書き留めたい**
**そうすることで** AIが自分に合った習慣を提案してくれる

**関連要件**: REQ-201, REQ-203

**詳細シナリオ**:
1. オンボーディング画面で「1年後の自分を自由に書いてください」と表示
2. テキストエリアに自由入力（例: 「毎朝6時に起きて、英語とランニングを続けられている自分」）
3. 「AIに相談する」をクリック
4. AIがWanna Beを分析し、長期目標の候補（最大3個）を提案
5. ユーザーが提案を承認または編集して保存

**前提条件**:
- ソーシャルログイン完了済み
- Claude APIが利用可能

**制約事項**:
- AIはWanna Beのテキストのみを送信し、個人情報（メールアドレス等）は送信しない

**優先度**: Must Have

## エピック2: 毎日の習慣トラッキング

### ストーリー 2.1: チェックリストで習慣を消し込む 🔵

**信頼性**: 🔵 *MVP機能選択・ヒアリングQ7より*

**私は** 毎朝アプリを開くビジネスパーソン **として**
**今日のルーティンをチェックリストで確認・完了登録したい**
**そうすることで** 習慣の実行を可視化し達成感を得られる

**関連要件**: REQ-404, REQ-501, REQ-205

**詳細シナリオ**:
1. ダッシュボードを開くと今日のルーティンリストが表示される
2. 各習慣の横に「今日の行動がどのWanna Beに効いているか」が表示される（例: 「→ 過去一の身体に +1」）
3. 習慣を実行したらチェックボックスをクリック
4. チェックが入るとストリークカウンターが更新される

**前提条件**:
- ログイン済み
- 1件以上の習慣が登録されている

**優先度**: Must Have

---

### ストーリー 2.2: 音声で日報を報告する 🔵

**信頼性**: 🔵 *ヒアリングQ7: 汎用音声入力方針より*

**私は** 移動中のビジネスパーソン **として**
**音声で今日の習慣達成状況を報告したい**
**そうすることで** タイピングなしに記録を残せる

**関連要件**: REQ-401, REQ-402, REQ-403

**詳細シナリオ**:
1. 音声入力ボタンをタップ（またはSiriで文字起こししてテキストを貼り付け）
2. 「早起き達成、筋トレできなかった、英語30分やった」と話す
3. AIが内容を解析し、各習慣の達成/未達成を自動判定
4. 「筋トレ❌」の場合、「できなかった理由を1行で教えてください」という入力欄が表示される
5. ユーザーが確認し「保存」をタップ

**前提条件**:
- ログイン済み
- Claude APIが利用可能

**優先度**: Must Have

---

### ストーリー 2.3: 未達成の理由を記録する 🔵

**信頼性**: 🔵 *ヒアリングQ3: 「できなかった理由」記録より*

**私は** 習慣を途切らせてしまったユーザー **として**
**できなかった理由を1行で記録したい**
**そうすることで** AIが翌週にパターンを分析して改善提案をしてくれる

**関連要件**: REQ-406, REQ-602

**詳細シナリオ**:
1. 習慣を❌でチェック（または音声で未達成と報告）
2. 「できなかった理由（任意）」入力欄が表示される
3. 「残業で時間がなかった」などを入力して保存
4. データが蓄積され、週次レビュー時にAIが分析

**優先度**: Must Have

---

## エピック3: ストリーク・達成可視化

### ストーリー 3.1: ストリークを確認する 🔵

**信頼性**: 🔵 *ヒアリングQ3: ストリーク可視化より*

**私は** 継続を頑張っているユーザー **として**
**今日で何日連続達成しているかを確認したい**
**そうすることで** ゲーム感覚で継続意欲が高まる

**関連要件**: REQ-502, REQ-503

**詳細シナリオ**:
1. ダッシュボードで各習慣の連続達成日数が表示される
2. 「ランニング 🔥14日連続」のように視覚的に表示
3. 途切れたらカウンターがリセットされ「0日」に戻る

**優先度**: Must Have

---

### ストーリー 3.2: 達成バッジを獲得する 🔵

**信頼性**: 🔵 *ヒアリングQ5: 達成バッジ・報酬より*

**私は** マイルストーンを達成したユーザー **として**
**バッジが付与されたことを知りたい**
**そうすることで** 小さな成功体験として達成感を得られる

**関連要件**: REQ-901, REQ-902

**詳細シナリオ**:
1. 習慣を7日連続達成すると「7日連続達成バッジ」が付与される
2. 画面にバッジ獲得の通知が表示される
3. プロフィール画面でこれまで獲得したバッジ一覧を確認できる

**優先度**: Should Have

## エピック4: AIコーチ体験

### ストーリー 4.1: AIから定期フィードバックを受け取る 🔵

**信頼性**: 🔵 *ヒアリングQ5: AIによる刺激・フィードバックより*

**私は** 習慣を継続しているユーザー **として**
**AIからパーソナライズされたフィードバックを受けたい**
**そうすることで** 自分では気づけなかったパターンや改善点を知れる

**関連要件**: REQ-601, REQ-602, REQ-604, REQ-605

**詳細シナリオ**:
1. 週次レビューが自動トリガーされる（毎週金曜または日曜）
2. AIが今週のデータを分析（達成率・ストリーク・未達成理由のパターン）
3. フィードバック例: 「筋トレの未達成が水曜・木曜に集中しています。夜残業が多い曜日のようです。朝のルーティンに移動しませんか？」
4. AIが改善提案として「習慣の時間帯を変更する」提案ボタンを表示
5. ユーザーが承認するとルーティンが更新される

**制約事項**:
- AIによるルーティン変更は時間帯変更・追加・削除のみ

**優先度**: Must Have

---

### ストーリー 4.2: Wanna Be を少しずつ磨く 🔵

**信頼性**: 🔵 *ヒアリングQ6より*

**私は** 目標が少しずつ具体化してきたユーザー **として**
**Wanna Be を更新してAIに再提案してもらいたい**
**そうすることで** 成長に合わせて習慣設計を見直せる

**関連要件**: REQ-202, REQ-203, REQ-204

**詳細シナリオ**:
1. Wanna Be画面を開き「編集」をクリック
2. テキストを修正（例: 「英語でプレゼンできる自分」を追加）
3. AIが更新内容を分析し「目標が4個になりました。3個以内に絞ることを推奨します」と提案
4. ユーザーが整理して保存

**優先度**: Must Have

---

## エピック5: 週次レビュー

### ストーリー 5.1: 週次レビューで振り返る 🔵

**信頼性**: 🔵 *ヒアリングQ3: 週次レビュー自動トリガーより*

**私は** 週末にレビューをするユーザー **として**
**AIが今週のデータを整理したレポートを見たい**
**そうすることで** 自分の成長と課題を客観的に把握できる

**関連要件**: REQ-701, REQ-702

**詳細シナリオ**:
1. 毎週金曜または日曜に通知「今週の振り返りをしましょう」が届く
2. 週次レビュー画面が開く
3. 今週の習慣達成率・ストリーク状況・できなかった理由のサマリーが表示される
4. AIコメント: 「今週は7習慣中6習慣を達成。筋トレが3回途切れましたが、理由に「残業」が多い傾向です」
5. 来週へのアドバイスが3行で表示される

**優先度**: Must Have
