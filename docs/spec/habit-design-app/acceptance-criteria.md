## REQ-101/102: ソーシャルログイン 🔵

**信頼性**: 🔵 *ヒアリングQ4より*

### Given
- ユーザーがトップページにアクセスしている

### When
- 「Googleでログイン」または「Appleでログイン」ボタンをクリックした

### Then
- OAuth認証フローが開始され、認証成功後にダッシュボードへ遷移する
- 初回ログイン時はWanna Be設定画面へ遷移する
- ユーザーIDとプロフィール情報がDBに保存される

### テストケース

#### 正常系
- [ ] **TC-101-01**: Googleアカウントで初回ログイン 🔵
  - **入力**: 有効なGoogleアカウント
  - **期待結果**: Wanna Be設定画面へ遷移
- [ ] **TC-101-02**: Googleアカウントで2回目ログイン 🔵
  - **入力**: 既存ユーザーのGoogleアカウント
  - **期待結果**: ダッシュボードへ遷移

#### 異常系
- [ ] **TC-101-E01**: 認証をキャンセルした場合 🟡
  - **入力**: OAuth画面でキャンセル
  - **期待結果**: トップページに戻り「ログインがキャンセルされました」と表示

## REQ-201/202/203: Wanna Be設定 🔵

**信頼性**: 🔵 *ヒアリングQ6より*

### Given
- ユーザーがログイン済み
- Wanna Be設定画面を開いている

### When
- テキストエリアに将来像を入力し「AIに相談する」をクリックした

### Then
- AIが入力内容を分析し、長期目標の候補（最大3個）を提案する
- ユーザーが承認または編集して保存できる
- 保存後、ダッシュボードに「今日の行動がどのWanna Beに効いているか」が表示される

### テストケース

#### 正常系
- [ ] **TC-201-01**: Wanna Be を初期登録 🔵
  - **入力**: 200文字以内の将来像テキスト
  - **期待結果**: AIが3個以内の目標候補を提案
- [ ] **TC-201-02**: Wanna Be を編集・更新 🔵
  - **入力**: 既存Wanna Beを一部修正
  - **期待結果**: 更新内容が保存され、AIが再分析を実施

#### 異常系
- [ ] **TC-201-E01**: AIが利用不可の場合 🔵
  - **入力**: Claude API接続エラー発生中
  - **期待結果**: 「AI提案は現在利用できません。テキストの保存のみ行います」と表示
- [ ] **TC-201-E02**: 長期目標が3個超えの場合 🔵
  - **入力**: 4個以上の目標を含むWanna Beテキスト
  - **期待結果**: AIが「目標を3個以内に絞ることを推奨します」と提案

## REQ-402/403: AI自動分類（音声入力） 🔵

**信頼性**: 🔵 *ヒアリングQ7: 汎用音声入力要件より*

### Given
- ユーザーがログイン済み
- 音声入力ボタンから入力した（またはテキストを貼り付けた）

### When
- 「早起き達成、筋トレできなかった、英語30分やった」などのテキストを送信した

### Then
- AIが内容を解析し、チェックリスト消込・ジャーナリング・日報などに自動分類する
- 対応する習慣の達成/未達成ステータスが更新される
- AIが判断できない場合は「どの操作ですか？」と確認ダイアログを表示する

### テストケース

#### 正常系
- [ ] **TC-402-01**: 明確な達成報告の自動分類 🔵
  - **入力**: 「今日は朝6時に起きてランニング5km完了」
  - **期待結果**: 「早起き」「ランニング」の達成ステータスが✅に更新
- [ ] **TC-402-02**: 混在した報告の分類 🔵
  - **入力**: 「英語は30分やった。筋トレは残業で無理だった」
  - **期待結果**: 英語✅、筋トレ❌に更新。筋トレに「できなかった理由」入力欄が表示

#### 異常系
- [ ] **TC-402-E01**: 分類不能な内容 🔵
  - **入力**: 「今日はなんかだるかった」
  - **期待結果**: 「どの操作ですか？（ジャーナリング保存 / 習慣のチェック / キャンセル）」確認ダイアログ表示

## REQ-502/503: ストリーク計算 🔵

**信頼性**: 🔵 *ヒアリングQ3: ストリーク可視化より*

### Given
- ユーザーが習慣を複数日連続して達成済み

### When
- ダッシュボードを表示した

### Then
- 各習慣に連続達成日数が「🔥N日連続」形式で表示される
- 当日未達成（❌）を記録すると、翌日以降のカウントが0にリセットされる

### テストケース

#### 正常系
- [ ] **TC-502-01**: 連続達成日数の表示 🔵
  - **入力**: 14日連続達成中の習慣
  - **期待結果**: 「🔥14日連続」が表示
- [ ] **TC-502-02**: ストリークのリセット 🔵
  - **入力**: 15日目に❌を記録
  - **期待結果**: 16日目以降のカウントが0にリセット

#### 境界値
- [ ] **TC-502-B01**: タイムゾーン考慮 🟡
  - **入力**: 深夜0時をまたいだ達成記録
  - **期待結果**: ユーザーのタイムゾーンに基づいて日付を判定
- [ ] **TC-502-B02**: 1日目のストリーク 🔵
  - **入力**: 初めて習慣を達成
  - **期待結果**: 「🔥1日連続」と表示

## REQ-601/602: AIフィードバック 🔵

**信頼性**: 🔵 *ヒアリングQ5: AIによる刺激・フィードバックより*

### Given
- ユーザーが1週間以上のトラッキングデータを保有
- 週次レビューがトリガーされた（または手動開始）

### When
- 週次レビュー画面を開いた

### Then
- AIが達成率・ストリーク・未達成理由のパターンを分析
- 具体的な改善提案を3行以内で表示
- 必要に応じて「習慣の時間帯変更」「習慣の削除」などの操作ボタンを提示

### テストケース

#### 正常系
- [ ] **TC-601-01**: 週次フィードバック生成 🔵
  - **入力**: 7日間のトラッキングデータ（達成率70%）
  - **期待結果**: 未達成パターンを含む改善提案が生成される（30秒以内）
- [ ] **TC-601-02**: AIが習慣変更を提案 🔵
  - **入力**: 特定曜日に習慣未達成が集中するデータ
  - **期待結果**: 時間帯変更の提案ボタンが表示される

#### 異常系
- [ ] **TC-601-E01**: API利用不可時のフォールバック 🔵
  - **入力**: Claude API接続エラー
  - **期待結果**: 「AIフィードバックは現在利用できません。集計データのみ表示します」とダッシュボードは通常表示

## REQ-303: AI変更の範囲制限 🔵

**信頼性**: 🔵 *ヒアリングQ9より*

### Given
- AIが習慣の変更を提案している

### When
- ユーザーがAI提案の変更を承認した

### Then
- 習慣の時間帯変更・追加・削除のみが実行される
- その他の変更（習慣の内容そのものの書き換え、UIカスタマイズ等）は実行されない

### テストケース

#### 正常系
- [ ] **TC-303-01**: 時間帯変更の承認 🔵
  - **入力**: 「筋トレを夜→朝7時に変更」の提案を承認
  - **期待結果**: 習慣の実行時刻が朝7時に更新される
- [ ] **TC-303-02**: 習慣追加の承認 🔵
  - **入力**: 「水を2L飲む習慣を追加」の提案を承認
  - **期待結果**: 新しい習慣がルーティンリストに追加される

#### 異常系
- [ ] **TC-303-E01**: 範囲外の変更試行（システム内部テスト） 🔵
  - **内容**: AIが範囲外の操作を試みた場合
  - **期待結果**: バリデーションエラーとなり変更は実行されない
