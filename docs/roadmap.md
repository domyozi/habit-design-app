## Phase 1: コア習慣トラッキング（現スコープ）

**期間**: 21日（168時間）

### 主要機能

- Wanna Be 設定 + Claude AI による習慣候補提案（SSEストリーミング）
- 習慣チェックリスト・ストリーク・バッジ
- **デイリーハブ（`/journal`）**: 音声/テキスト一入力で朝の意図設定・夜の振り返りを自動判別
- 週次レビュー + AI フィードバック
- 習慣トラッキング可視化（達成率グラフ・カレンダー）

### デイリーハブ設計原則

- 単一入力フィールド（切り替えなし）
- 時刻 + 今日のログ状況をコンテキストとしてAIが解釈
- 対応アクション: 習慣ログ更新・未達成理由記録・3行日報生成
- EDGE-001 準拠: AI 障害時もログは手動で記録可能

## Phase 2: 外部サービス連携エージェント（将来）

**前提**: Phase 1 が安定稼働してから着手

### 設計方針

デイリーハブの入力を**MCP (Model Context Protocol) 経由のエージェント**として拡張する。
音声で話しかけると Claude が複数のツールを呼び出して一括処理する形式。

### 対応予定の外部サービス

| サービス | 連携内容 | 対応アクション |
|---------|---------|--------------|
| **Todoist / TickTick** | タスク管理連携 | タスク完了記録・新規タスク追加・優先度変更 |
| **Google Calendar** | カレンダー連携 | 習慣の時間ブロック作成・予定確認・スケジュール最適化 |
| **Notion / Obsidian** | ノート連携 | 日報の自動保存・週次レビューのログ出力 |

### 技術方針

```
音声入力
    ↓
Claude (MCP クライアント)
    ├── mcp-todoist: タスク操作
    ├── mcp-google-calendar: カレンダー操作
    ├── mcp-habit-tracker: 習慣ログ更新（habit-design-app 自体）
    └── mcp-notes: 日報・ノート保存
```

- 各 MCP サーバーは独立して実装・テスト可能
- ユーザーが「どのサービスを連携するか」を設定画面で選択
- プライバシー設計: Claude に送信するのはユーザーが許可したデータのみ
- REQ-605 拡張: 外部サービスデータの送信ルールを明文化

### 設定画面への追加項目

- 連携サービス管理（OAuth 認証 / 切断）
- 各サービスへの送信データの範囲設定
- エージェントアクションの承認モード（自動/確認要）

### 実装優先順位

1. Google Calendar（予定確認 → 習慣時間ブロック）
2. Todoist（タスク完了 → 習慣ログ連動）
3. Notion（日報自動保存）

## 設計上の制約（Phase 2 以降の注意点）

### セキュリティ
- 外部サービスの OAuth トークンは Railway の環境変数で管理（クライアント露出禁止）
- Claude に送信する外部データはユーザーが明示的に許可したもののみ
- 外部サービスへの書き込み操作は「確認後実行」モードをデフォルトに

### スコープ管理
- Phase 2 の追加タスクは `TASK-0026` 以降に採番
- 外部連携ごとに独立したタスクファイルを作成
- `docs/tasks/habit-design-app/overview.md` のタスク番号管理を更新すること
