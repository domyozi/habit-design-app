# オンボーディング

初回ログイン後に表示されるWanna Be（なりたい自分）設定画面。ユーザーが将来像をテキストで入力し、AIが長期目標候補（最大3件）をストリーミング提案する。ユーザーが承認するとダッシュボードへ遷移する。

## 画面項目

| # | 項目名 | 種別 | 必須 | 説明 |
|---|--------|------|------|------|
| 1 | Wanna Be テキストエリア | textarea | Yes | 「なりたい自分」を自由記述で入力（200文字以内） <!-- 🔵 --> |
| 2 | 「AIに相談する」ボタン | button | - | SSEストリーミングによるAI分析を開始する <!-- 🔵 --> |
| 3 | AIストリーミング表示エリア | text (display) | - | AI応答をリアルタイムで逐次表示するエリア <!-- 🔵 --> |
| 4 | 目標候補リスト | list | - | AIが提案する長期目標候補（最大3件） <!-- 🔵 --> |
| 5 | 各目標の「承認」ボタン | button | - | 対象目標・習慣をDBに保存してダッシュボードへ遷移 <!-- 🔵 --> |

## バリデーション

| 項目 | ルール | エラーメッセージ |
|------|--------|-----------------|
| Wanna Be テキストエリア | 必須入力 | Wanna Beを入力してください <!-- 🔵 --> |
| Wanna Be テキストエリア | 200文字以内 | 200文字以内で入力してください <!-- 🟡 --> |

## インタラクション

### AI分析開始 <!-- 🔵 -->
- Wanna Be 入力後「AIに相談する」ボタン押下 → バックエンド SSE エンドポイント接続 → AI（Claude）が分析 → ストリーミング表示エリアにリアルタイム出力

### 目標候補の表示 <!-- 🔵 -->
- AIストリーミング完了 → 長期目標候補リスト（最大3件）が表示される

### 目標の承認・保存 <!-- 🔵 -->
- 「承認」ボタン押下 → 目標・関連習慣を Supabase DB に保存 → `/` (ダッシュボード) へ遷移

### AI障害時のフォールバック <!-- 🔵 -->
- Claude API 接続エラー発生 → 「AIは利用できません。テキストのみ保存されました」メッセージ表示 → Wanna Beテキストのみ保存して続行可能

### 目標提案が3件超えの場合 <!-- 🔵 -->
- AIが「目標を3個以内に絞ることを推奨します」と提案メッセージを表示

## 状態

| 状態 | 条件 | 表示内容の変化 |
|------|------|---------------|
| 初期表示 | 画面読み込み時 | Wanna Be テキストエリアのみ表示、ストリーミングエリアと目標候補リストは非表示 <!-- 🔵 --> |
| AI生成中 | 「AIに相談する」押下後 | ストリーミングテキストをリアルタイム表示、ボタン非活性 <!-- 🟡 --> |
| AI生成完了 | SSEストリーム終了後 | 目標候補リスト（最大3件）と各承認ボタンが表示される <!-- 🔵 --> |
| エラー（AI障害） | Claude API接続エラー時 | 「AIは利用できません。テキストのみ保存されました」メッセージを表示 <!-- 🔵 --> |
| 保存中 | 「承認」ボタン押下後 | ボタン非活性、スピナー表示 <!-- 🔴 --> |

## 備考

- 本画面は初回ログイン時のみ表示される。既存ユーザーがアクセスした場合はダッシュボードへリダイレクト <!-- 🟡 -->
- AI分析はバックエンドの FastAPI (`/ai_coach`) 経由で Claude API を呼び出す（クライアントに API キーを露出させない）
- SSE接続は `frontend/src/lib/api.ts` の共通関数で管理する
- Wanna Be テキストは `/wanna-be` 画面でも後から編集可能
- AI提案の習慣変更範囲は「追加・削除・時間帯変更」に限定される（REQ-303）
