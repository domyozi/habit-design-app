# ダッシュボード

ログイン後のメイン画面。今日の習慣チェックリストとストリーク確認、週次統計を一覧でき、デイリーハブや各機能への起点となる。

## 画面項目

| # | 項目名 | 種別 | 必須 | 説明 |
|---|--------|------|------|------|
| 1 | 今日の習慣チェックリスト | list | - | 各習慣の名前・チェックボックス・ストリーク日数を表示 <!-- 🔵 --> |
| 2 | 習慣チェックボックス | checkbox | - | 各習慣の達成・未達成をトグルする <!-- 🔵 --> |
| 3 | ストリーク表示 | text | - | 各習慣に「🔥N日連続」形式で連続達成日数を表示 <!-- 🔵 --> |
| 4 | 未達成理由入力欄 | textarea | No | チェックボックスOFF時に展開される任意入力欄 <!-- 🔵 --> |
| 5 | 週次統計サマリー | display | - | 今週の習慣達成率をサマリー表示 <!-- 🔵 --> |
| 6 | デイリーハブへのボタン | button | - | /journal へ遷移するナビゲーション <!-- 🔵 --> |
| 7 | バッジ獲得通知 | popup | - | 7日連続達成時にバッジ獲得をポップアップ表示 <!-- 🔵 --> |
| 8 | Wanna Be設定リンク | link | - | /wanna-be へ遷移 <!-- 🟡 --> |
| 9 | 週次レビューリンク | link | - | /weekly-review へ遷移 <!-- 🟡 --> |
| 10 | トラッキングリンク | link | - | /tracking へ遷移 <!-- 🟡 --> |
| 11 | 設定リンク | link | - | /settings へ遷移 <!-- 🟡 --> |

## バリデーション

| 項目 | ルール | エラーメッセージ |
|------|--------|-----------------|
| 未達成理由 | 任意入力（空可） | - <!-- 🔵 --> |

## インタラクション

### 習慣達成記録 <!-- 🔵 -->
- チェックボックスON → `habit_log` 保存 → ストリーク日数更新 → バッジ付与チェック実行
- 7日連続達成が確認された場合 → バッジ獲得通知ポップアップ表示

### 習慣未達成記録 <!-- 🔵 -->
- チェックボックスOFF → 未達成理由入力欄が展開 → 理由入力（任意）→ 保存

### デイリーハブ遷移 <!-- 🔵 -->
- 「デイリーハブ」ボタン押下 → /journal へ遷移

### ナビゲーション <!-- 🟡 -->
- Wanna Be設定リンク → /wanna-be へ遷移
- 週次レビューリンク → /weekly-review へ遷移
- トラッキングリンク → /tracking へ遷移
- 設定リンク → /settings へ遷移

## 状態

| 状態 | 条件 | 表示内容の変化 |
|------|------|---------------|
| ローディング | 習慣一覧取得中 | スケルトン表示 <!-- 🔵 --> |
| 通常 | 習慣一覧取得完了 | 習慣チェックリスト・ストリーク・週次統計を表示 <!-- 🔵 --> |
| 全達成 | 全習慣チェック済み | 全チェックボックスがON状態 <!-- 🔵 --> |
| 未達成理由入力中 | チェックボックスOFFのとき | 対象習慣の下に理由入力欄が展開 <!-- 🔵 --> |
| バッジ通知 | 7日連続達成トリガー | バッジ獲得ポップアップを前面に表示 <!-- 🔵 --> |

## 備考

- ストリーク表示はタイムゾーン考慮が必要（TC-502-B01）<!-- 🟡 -->
- バッジ獲得通知は7日連続達成をトリガーとする（REQ-502/503 準拠）<!-- 🔵 -->
- AI障害時はダッシュボードの手動チェックがフォールバックとなる（EDGE-001）<!-- 🔴 -->
- チェックボックス操作からUI更新まで2秒以内（NFR-001 パフォーマンス要件）<!-- 🟡 -->
