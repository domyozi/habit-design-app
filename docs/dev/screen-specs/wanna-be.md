# Wanna Be 設定

将来像（Wanna Be）を入力・編集し、AIに分析を依頼して長期目標候補を提案してもらう画面。AI提案の承認・却下操作を通じてGoalをDBに保存する。

## 画面項目

| # | 項目名 | 種別 | 必須 | 説明 |
|---|--------|------|------|------|
| 1 | Wanna Be テキストエリア | textarea | Yes | 「なりたい自分」を入力・編集する。既存設定値を初期表示 <!-- 🔵 --> |
| 2 | 「AIに相談する」ボタン | button | - | クリックでSSE接続を開始しAI分析を実行 <!-- 🔵 --> |
| 3 | AIストリーミング表示エリア | display area | - | AIの分析・提案テキストをストリーミング表示する <!-- 🔵 --> |
| 4 | 目標候補リスト | list | - | AIが提案する長期目標の一覧（最大3件） <!-- 🔵 --> |
| 5 | 各目標の「承認」ボタン | button | - | 該当目標をDBに保存する <!-- 🔵 --> |
| 6 | 各目標の「却下」ボタン | button | - | 該当目標を提案リストから除外する <!-- 🔵 --> |
| 7 | AI障害メッセージ | text | - | AI利用不可時に表示するフォールバックメッセージ <!-- 🔵 --> |

## バリデーション

| 項目 | ルール | エラーメッセージ |
|------|--------|-----------------|
| Wanna Be テキストエリア | 必須入力 | 「なりたい自分」を入力してください <!-- 🔵 --> |
| Wanna Be テキストエリア | 200文字以内 | 200文字以内で入力してください <!-- 🟡 --> |

## インタラクション

### Wanna Be 編集・AI分析開始 <!-- 🔵 -->
- テキストエリアを編集 → 「AIに相談する」ボタン押下 → SSE接続開始 → AIがストリーミングで目標候補を表示

### 目標候補の承認 <!-- 🔵 -->
- 「承認」ボタン押下 → 対象目標をDBに保存 → 承認済み状態に更新

### 目標候補の却下 <!-- 🔵 -->
- 「却下」ボタン押下 → 対象目標を提案リストから除外（DBには保存しない）

### AI障害時のフォールバック <!-- 🔵 -->
- SSE接続失敗またはAPIエラー → 「AIは利用できません。テキストの保存のみ行います」メッセージ表示 → テキスト保存は可能

### 目標確認画面への遷移 <!-- 🟡 -->
- 承認後、goals画面へのナビゲーションリンクを表示

## 状態

| 状態 | 条件 | 表示内容の変化 |
|------|------|---------------|
| 初期表示 | ページ読み込み時 | 既存Wanna Be値がテキストエリアに表示。AI提案エリアは非表示 <!-- 🔵 --> |
| AI生成中 | SSE接続開始後 | ストリーミングテキストがリアルタイムで表示。「AIに相談する」ボタンdisabled <!-- 🔵 --> |
| AI生成完了 | SSEストリーム終了後 | 目標候補リストが表示。承認・却下ボタンが活性 <!-- 🔵 --> |
| AI障害 | Claude API接続エラー時 | 「AIは利用できません。テキストの保存のみ行います」メッセージ表示 <!-- 🔵 --> |
| テキスト保存中 | AI障害時の保存処理中 | 保存ボタンdisabled + スピナー表示 <!-- 🔴 --> |

## 備考

- AC: REQ-201/202/203（Wanna Be設定・AI提案・承認フロー）
- AI提案は最大3件に制限。4件以上が候補になった場合はAIが「3個以内に絞ることを推奨します」と提示（TC-201-E02）
- SSE（Server-Sent Events）でFastAPIからClaudeのストリーミング応答をリレーする
- 画面遷移: `/wanna-be` → `/goals`（目標確認）、承認完了後 → `/`（ダッシュボード）
