# 週次レビュー

週次レビューを手動または自動でトリガーし、AIが達成率・ストリーク・未達成理由のパターンを分析して改善提案をストリーミング表示する画面。ユーザーはAI提案を承認して習慣の変更（時間帯変更・追加・削除）を実行できる。

## 画面項目

| # | 項目名 | 種別 | 必須 | 説明 |
|---|--------|------|------|------|
| 1 | 「週次レビューを開始する」ボタン | button | - | SSE接続を開始しAIフィードバック生成を起動 <!-- 🔵 --> |
| 2 | AIフィードバックストリーミング表示エリア | display area | - | AIの分析・改善提案テキストをリアルタイム表示 <!-- 🔵 --> |
| 3 | AI提案リスト | list | - | 時間帯変更・習慣追加・習慣削除などの具体的な変更提案（最大3件） <!-- 🔵 --> |
| 4 | 各提案の「承認」ボタン | button | - | 該当提案を実行し習慣データを更新 <!-- 🔵 --> |
| 5 | AI障害メッセージ | text | - | AI利用不可時のフォールバックメッセージ <!-- 🔵 --> |
| 6 | 週間達成率サマリー | statistics display | - | 今週の全体達成率を数値で表示 <!-- 🟡 --> |
| 7 | 未達成パターン表示 | text/chart | - | 未達成が集中する曜日・時間帯などのパターン表示 <!-- 🟡 --> |

## バリデーション

| 項目 | ルール | エラーメッセージ |
|------|--------|-----------------|
| なし | - | この画面にバリデーション対象のフォーム入力はない <!-- 🔵 --> |

## インタラクション

### 週次レビュー開始 <!-- 🔵 -->
- 「週次レビューを開始する」ボタン押下 → SSE接続開始 → AIが達成率・ストリーク・未達成パターンを分析 → フィードバックテキストをストリーミング表示 → 完了後に提案リスト表示

### AI提案の承認 <!-- 🔵 -->
- 「承認」ボタン押下 → 習慣の `scheduled_time` / 有効状態を更新 → 承認済み状態に変更

### 習慣時間帯変更提案の承認 <!-- 🔵 -->
- 時間帯変更提案を承認 → 該当習慣の実行時刻を更新（例: 筋トレを夜→朝7時に変更）

### 習慣追加提案の承認 <!-- 🔵 -->
- 習慣追加提案を承認 → 新しい習慣をルーティンリストに追加

### 習慣削除提案の承認 <!-- 🔵 -->
- 習慣削除提案を承認 → 対象習慣をルーティンリストから削除

### AI変更の範囲制限 <!-- 🔵 -->
- AIが実行できる変更は「時間帯変更・追加・削除」のみ。習慣内容の書き換えやUIカスタマイズは実行されない

### AI障害時のフォールバック <!-- 🔵 -->
- SSE接続失敗またはAPIエラー → 「AIフィードバックは現在利用できません。集計データのみ表示します」メッセージ表示 → 週間達成率サマリーは表示継続

## 状態

| 状態 | 条件 | 表示内容の変化 |
|------|------|---------------|
| 初期表示 | ページ読み込み時 | 「週次レビューを開始する」ボタンのみ表示 <!-- 🔵 --> |
| レビュー中 | SSE接続開始後 | AIフィードバックがストリーミングで逐次表示。開始ボタンdisabled <!-- 🔵 --> |
| レビュー完了 | SSEストリーム終了後 | AI提案リストが表示。各提案の「承認」ボタンが活性 <!-- 🔵 --> |
| 提案承認済み | 承認ボタン押下後 | 承認済み提案をvisually無効化（グレーアウト等） <!-- 🟡 --> |
| AI障害 | Claude API接続エラー時 | 「AIフィードバックは現在利用できません。集計データのみ表示します」メッセージ + 達成率サマリー表示 <!-- 🔵 --> |
| データ不足 | トラッキングデータ1週間未満 | 「データが不足しています。1週間以上記録するとレビューが開始できます」メッセージ表示 <!-- 🔴 --> |

## 備考

- AC: REQ-601/602（AIフィードバック）、REQ-303（AI変更の範囲制限）
- AIフィードバックは30秒以内に生成完了することが目標（TC-601-01）
- 週次レビューはAPSchedulerによる自動トリガー（設定画面で曜日指定）と手動開始の両方をサポート
- AI提案の変更範囲は「追加・削除・時間帯変更」の3種類に限定。範囲外の変更試みはバリデーションエラー（REQ-303）
- 画面遷移: `/` → `/weekly-review`（ダッシュボードから）、`/weekly-review` → `/weekly-review`（AI提案承認後に同画面更新）
