# ログイン

ユーザーがGoogle/AppleアカウントによるOAuth認証でアプリにログインする画面。初回ログイン時はオンボーディング画面へ、既存ユーザーはダッシュボードへ遷移する。

## 画面項目

| # | 項目名 | 種別 | 必須 | 説明 |
|---|--------|------|------|------|
| 1 | アプリ説明・キャッチコピー | text (display) | - | アプリの概要説明テキスト <!-- 🟡 --> |
| 2 | Googleでログイン | button | Yes | SupabaseのGoogle OAuthフローを開始する <!-- 🔵 --> |
| 3 | Appleでログイン | button | Yes | SupabaseのApple OAuthフローを開始する <!-- 🔵 --> |

## バリデーション

| 項目 | ルール | エラーメッセージ |
|------|--------|-----------------|
| OAuthフロー | 認証キャンセル時 | ログインがキャンセルされました <!-- 🟡 --> |
| OAuthフロー | 認証失敗時 | 認証に失敗しました。再度お試しください <!-- 🔴 --> |

## インタラクション

### Googleログイン成功（初回） <!-- 🔵 -->
- 「Googleでログイン」ボタン押下 → Supabase OAuth フロー開始 → 認証完了 → `is_new_user` 判定 → `/onboarding` へリダイレクト

### Googleログイン成功（既存ユーザー） <!-- 🔵 -->
- 「Googleでログイン」ボタン押下 → Supabase OAuth フロー開始 → 認証完了 → `/` (ダッシュボード) へリダイレクト

### Appleログイン成功（初回） <!-- 🔵 -->
- 「Appleでログイン」ボタン押下 → Supabase OAuth フロー開始 → 認証完了 → `/onboarding` へリダイレクト

### Appleログイン成功（既存ユーザー） <!-- 🔵 -->
- 「Appleでログイン」ボタン押下 → Supabase OAuth フロー開始 → 認証完了 → `/` (ダッシュボード) へリダイレクト

### 認証キャンセル <!-- 🟡 -->
- OAuth画面でキャンセル → トップページに戻り「ログインがキャンセルされました」トースト表示

### 認証失敗 <!-- 🔵 -->
- OAuth エラー発生 → エラートースト表示 → ログイン画面に留まる

## 状態

| 状態 | 条件 | 表示内容の変化 |
|------|------|---------------|
| 初期表示 | ページ読み込み時 | ログインボタン2つ表示、アプリ説明テキスト表示 <!-- 🔵 --> |
| OAuthリダイレクト中 | ログインボタン押下後 | ローディング表示（ボタン非活性またはスピナー） <!-- 🟡 --> |
| 認証エラー | OAuth失敗時 | エラートースト表示 <!-- 🔵 --> |
| 認証キャンセル | OAuth画面でキャンセル時 | 「ログインがキャンセルされました」トースト表示 <!-- 🟡 --> |

## 備考

- バリデーションはSupabase OAuthに委任。フォーム入力フィールドは存在しない
- ユーザーIDとプロフィール情報はOAuth成功後にSupabase DBへ自動保存される
- 初回ログイン判定はSupabase Authのセッション情報またはユーザーレコードの存在確認で行う <!-- 🔴 -->
- SupabaseクライアントはReactフロントエンド（`src/lib/supabase.ts`）で初期化される
