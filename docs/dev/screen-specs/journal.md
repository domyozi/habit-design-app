# デイリーハブ

テキストまたは音声で一言入力するだけで、AIが時刻・ログ状況・入力内容を解析し、習慣チェック・理由記録・メモ保存などの複数アクションを一括実行する日次入力ハブ画面。朝/夜の切り替えUIは持たず、時刻に応じたコンテキストラベルで状況を示す。

## 画面項目

| # | 項目名 | 種別 | 必須 | 説明 |
|---|--------|------|------|------|
| 1 | コンテキストラベル | display | - | 時刻で自動表示。5:00〜11:59=🌅朝、12:00〜17:59=☀️昼、18:00〜23:59=🌙夜、0:00〜4:59=🌙深夜 <!-- 🔵 --> |
| 2 | 統合入力フィールド | textarea | Yes | テキスト入力と音声入力を単一フィールドで受け付ける。切り替えUIなし <!-- 🔵 --> |
| 3 | マイクボタン | button | - | 長押しで Web Speech API 起動。音声をリアルタイムにテキスト変換しフィールドへ反映 <!-- 🔵 --> |
| 4 | 送信ボタン | button | - | 入力内容をAIへ送信。空入力時は無効 <!-- 🔵 --> |
| 5 | AI解釈・実行結果エリア | display | - | AIが実行したアクション結果のフィードバック表示（例: 「✅ 英語30分 → 達成ログ記録」） <!-- 🔵 --> |
| 6 | 今日の習慣クイックサマリー | display | - | 読み取り専用の習慣一覧（小サイズ）。達成状況を一覧表示 <!-- 🔵 --> |
| 7 | アクション履歴 | list | - | 今日の操作ログ 最大5件を表示 <!-- 🔵 --> |
| 8 | 手動選択UI | dialog | - | AIが判断できない入力に対して表示。「ジャーナリング保存 / 習慣のチェック / キャンセル」を選択肢として提示 <!-- 🔵 --> |

## バリデーション

| 項目 | ルール | エラーメッセージ |
|------|--------|-----------------|
| 統合入力フィールド | 必須（空送信不可） | （送信ボタンを非活性にすることで防止） <!-- 🔵 --> |

## インタラクション

### 音声入力 <!-- 🔵 -->
- マイクボタン長押し → Web Speech API 起動 → リアルタイム文字起こし開始 → テキストフィールドに反映
- ボタン離す → 音声入力終了

### AI一括処理（送信） <!-- 🔵 -->
- 送信ボタン押下 → Claude（FastAPI経由）へリクエスト送信 → AI が時刻・今日のログ状況・入力内容を解析 → 複数アクションを一括実行 → フィードバック表示
- 解釈例:
  - 「英語30分できた」→ 英語習慣ログを達成（✅）に更新
  - 「筋トレはできなかった、残業で」→ 筋トレ未達成（❌）+ 理由「残業」を記録
  - 「今日は全部頑張る」→ 朝の意図としてメモ保存
  - 分類不能な内容 → 手動選択UI（確認ダイアログ）を表示

### 分類不能時の手動選択 <!-- 🔵 -->
- AI判断不能 → 「どの操作ですか？（ジャーナリング保存 / 習慣のチェック / キャンセル）」確認ダイアログ表示 → ユーザーが選択 → 選択に応じた処理を実行

### コンテキストラベル自動切替 <!-- 🔵 -->
- 画面表示時にデバイス時刻を取得し、以下の基準でラベルを決定:
  - 5:00〜11:59 → 🌅朝
  - 12:00〜17:59 → ☀️昼
  - 18:00〜23:59 → 🌙夜
  - 0:00〜4:59 → 🌙深夜

## 状態

| 状態 | 条件 | 表示内容の変化 |
|------|------|---------------|
| 待機中 | 画面表示時（入力前） | 空フィールド + コンテキストラベル表示。送信ボタン非活性 <!-- 🔵 --> |
| 音声入力中 | マイクボタン長押し中 | マイクアニメーション + リアルタイム文字起こしをフィールドに反映 <!-- 🔵 --> |
| AI処理中 | 送信後・AI応答待ち | スピナー表示。送信ボタン非活性 <!-- 🔵 --> |
| 完了 | AI処理成功 | アクション結果フィードバックを結果エリアに表示 <!-- 🔵 --> |
| 手動選択待ち | AI判断不能 | 確認ダイアログ（手動選択UI）を表示 <!-- 🔵 --> |
| AIエラー | Claude API接続不可 | 「AI解釈できません。手動で記録してください」メッセージ + ダッシュボードへのリンク表示 <!-- 🔵 --> |

## 備考

- 朝/夜の切り替えUIは持たない。コンテキストラベルで状況を示すだけで入力フォームは1つ <!-- 🔵 -->
- Web Speech API はChrome/Edge のみ対応。Firefox・Safari は未対応 <!-- 🟡 -->
- AI障害時はダッシュボードの手動チェックにフォールバック（EDGE-001 準拠）<!-- 🔵 -->
- Phase 2 拡張ポイント: 外部MCP連携時、AIが呼び出すツールを増やすだけでUI変更は不要な設計 <!-- 🔵 -->
- Claude API キーはバックエンド（FastAPI）のみで保持。クライアントには露出しない（NFR-101）<!-- 🔵 -->
