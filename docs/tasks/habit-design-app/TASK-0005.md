## 実装詳細

### 1. FastAPIアプリ設定（main.py） 🔵

**信頼性**: 🔵 *architecture.md より*

FastAPIアプリケーションのエントリポイント。CORS設定、ルーター組み込み、lifespanによるSupabaseクライアント初期化を実装する。

- CORS: `FRONTEND_URL` 環境変数で指定したオリジンを許可（デフォルト: `http://localhost:5173`）
- メソッド: `GET, POST, PATCH, DELETE, OPTIONS`
- ヘッダー: `Authorization, Content-Type`
- credentialsあり
- APIルーターは `/api` プレフィックスで組み込み
- lifespan コンテキストマネージャでSupabaseクライアントを初期化し、`app.state.supabase` に格納

**実装ファイル**: `backend/app/main.py`

---

### 2. Pydanticスキーマ定義（schemas.py） 🔵

**信頼性**: 🔵 *api-endpoints.md・interfaces.ts より*

全エンドポイントで使用するリクエスト/レスポンス型をPydanticモデルとして定義する。

#### ドメインモデル

| モデル名 | 主なフィールド |
|---|---|
| `UserProfile` | id, email, display_name, timezone, weekly_review_day, notification_email, notification_enabled, created_at |
| `WannaBe` | id, user_id, text, version, is_current, created_at |
| `Goal` | id, user_id, title, description, is_active, created_at |
| `Habit` | id, user_id, goal_id, title, scheduled_time, frequency, is_active, current_streak, longest_streak, wanna_be_connection_text, created_at |
| `HabitLog` | id, habit_id, user_id, log_date, completed, note, created_at |
| `FailureReason` | id, habit_id, user_id, log_date, reason, created_at |
| `JournalEntry` | id, user_id, content, input_type, created_at |
| `WeeklyReview` | id, user_id, week_start, achievement_rate, summary, actions, status, created_at |
| `BadgeDefinition` | id, name, description, condition_type, condition_value, icon_url |
| `UserBadge` | id, user_id, badge_id, habit_id, earned_at, badge |

#### リクエストモデル

| モデル名 | 主なフィールド |
|---|---|
| `CreateHabitRequest` | goal_id, title, scheduled_time, frequency |
| `UpdateHabitRequest` | title, scheduled_time, frequency, action |
| `UpdateHabitLogRequest` | completed, note, log_date |
| `VoiceInputRequest` | text, date |

#### 共通型

- `APIResponse[T]`: `data: T`, `message: str`, `success: bool` のジェネリックレスポンス型
- `AIActionType`: `Literal["change_time", "add_habit", "remove_habit"]`（REQ-303: 3種類のみ）
- `AIAction`: `action_type: AIActionType`, `habit_id: Optional[str]`, `params: dict`
- `ErrorResponse`: `error_code: str`, `message: str`, `detail: Optional[Any]`

**実装ファイル**: `backend/app/models/schemas.py`

---

### 3. ルーター統合（routes/__init__.py） 🔵

**信頼性**: 🔵 *architecture.md より*

各機能のルーターをまとめて `/api` プレフィックス配下に組み込む統合モジュール。後続タスクで追加されるルーターをここに登録する。

```
/api/users/me
/api/wanna-be
/api/goals
/api/habits
/api/voice-input
/api/ai/weekly-review/stream
/api/notifications/settings
```

**実装ファイル**: `backend/app/api/routes/__init__.py`

---

### 4. エラーハンドリングミドルウェア 🔵

**信頼性**: 🔵 *architecture.md より*

FastAPIデフォルトの422バリデーションエラーを日本語メッセージに変換するカスタム例外ハンドラーを登録する。

- `RequestValidationError` → 422、フィールド名と日本語エラーメッセージを含む `ErrorResponse`
- `HTTPException` → 対応するステータスコードと `ErrorResponse`
- 予期しない例外 → 500、`ErrorResponse(error_code="INTERNAL_SERVER_ERROR")`

**実装ファイル**: `backend/app/main.py`（例外ハンドラー登録）, `backend/app/core/exceptions.py`（カスタム例外クラス）

## 注意事項

- `AIActionType` は `change_time / add_habit / remove_habit` の3種類のみ（REQ-303）。スキーマレベルで制約をかけることで、後続APIの実装ミスを防ぐ
- Supabaseクライアントは lifespan で初期化し、`app.state.supabase` に格納する。各エンドポイントでは `Request` オブジェクト経由でアクセスする
- `APIResponse[T]` はジェネリック型として定義し、型安全なレスポンスを実現する
- CORS設定の `FRONTEND_URL` は環境変数から取得し、`.env` に定義する
