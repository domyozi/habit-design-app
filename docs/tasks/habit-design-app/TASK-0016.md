習慣を未達成にした際の理由入力フォームと、3行日報フォームを実装する。
未達成理由は任意入力で、AIが後日パターン分析に使用する（REQ-602）。

## 実装詳細

### 1. 未達成理由入力コンポーネント 🔵

**信頼性**: 🔵 *REQ-406・user-stories 2.3 より*

**実装ファイル**: `frontend/src/components/habits/FailureReasonInput.tsx`

- 習慣を❌にした後にインラインで表示
- React Hook Form でフォーム管理
- 「任意」ラベル付き（入力は強制しない）
- 送信: `POST /habits/{id}/failure-reason`
- 「スキップ」ボタンで入力欄を閉じる

---

### 2. 3行日報フォーム 🔵

**信頼性**: 🔵 *REQ-405 より*

**実装ファイル**: `frontend/src/components/dashboard/DailyReportForm.tsx`

3つの入力欄:
- **習慣達成状況**: チェックリストから自動集計（例: 「3/5達成」）、表示のみ
- **今日やったこと**: 1行テキスト入力（placeholder: 「今日の主な行動を1行で」）
- **明日のラスボス**: 1行テキスト入力（placeholder: 「明日最も重要なタスクは？」）

送信: journal_entriesに `type: daily_report` で保存
