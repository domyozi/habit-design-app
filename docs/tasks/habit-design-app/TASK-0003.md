## 実装詳細

### 1. Supabase プロジェクト作成 🔵

**信頼性**: 🔵 *architecture.md インフラ構成より*

1. [https://supabase.com](https://supabase.com) にアクセスしてサインイン
2. 「New project」をクリック
3. 以下の情報を入力してプロジェクトを作成する:
   - Project name: `habit-design-app`
   - Database password: 強力なパスワードを設定（安全な場所に保管）
   - Region: Northeast Asia (Tokyo) を選択
4. プロジェクト作成完了後、以下の情報を控える:
   - Project URL: `Settings > API > Project URL`
   - anon/public key: `Settings > API > Project API keys > anon public`
   - service_role key: `Settings > API > Project API keys > service_role`（秘密鍵・厳重管理）
   - JWT Secret: `Settings > API > JWT Settings > JWT Secret`

**実装ファイル**: （Supabase Webコンソール操作）

---

### 2. Google OAuth プロバイダー設定 🔵

**信頼性**: 🔵 *REQ-101 ソーシャルログイン要件より*

1. [Google Cloud Console](https://console.cloud.google.com) でプロジェクトを作成（または既存プロジェクトを使用）
2. 「APIとサービス > 認証情報」でOAuth 2.0クライアントIDを作成:
   - アプリケーションの種類: ウェブアプリケーション
   - 承認済みのリダイレクトURI: `https://<your-project-ref>.supabase.co/auth/v1/callback`
3. 取得した Client ID と Client Secret を控える
4. Supabase ダッシュボード `Authentication > Providers > Google` で有効化:
   - Client ID と Client Secret を入力して保存

**実装ファイル**: （Google Cloud Console・Supabase Webコンソール操作）

---

### 3. Apple OAuth プロバイダー設定 🔵

**信頼性**: 🔵 *REQ-101 ソーシャルログイン要件より*

1. [Apple Developer](https://developer.apple.com) でApp IDとService IDを作成
2. Sign in with Apple の設定:
   - Services ID の Domains and Subdomains に Supabase プロジェクトドメインを追加
   - Return URLs に `https://<your-project-ref>.supabase.co/auth/v1/callback` を追加
3. 秘密鍵（.p8ファイル）を生成してダウンロード
4. Supabase ダッシュボード `Authentication > Providers > Apple` で有効化:
   - Services ID、Team ID、Key ID、秘密鍵の内容を入力して保存

**実装ファイル**: （Apple Developer Console・Supabase Webコンソール操作）

### 5. RLS ポリシー確認 🔵

**信頼性**: 🔵 *NFR-102 RLS設計より*

全テーブルで RLS（Row Level Security）が有効になっていることを確認する。

Supabase ダッシュボード `Authentication > Policies` で各テーブルの RLS ポリシーを確認する。確認コマンド（SQL Editor で実行）:

```sql
SELECT tablename, rowsecurity
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY tablename;
```

全テーブルの `rowsecurity` が `true` であることを確認する。

**実装ファイル**: （Supabase SQL Editor で確認）

### 6. 認証ユーザー作成トリガー設定 🔵

**信頼性**: 🔵 *database-schema.sql・REQ-101 ユーザー管理要件より*

Supabase SQL Editor で以下を実行する。

```sql
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.user_profiles (id)
  VALUES (NEW.id);
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
```

このトリガーにより、Supabase Auth でユーザーが作成されると自動的に `public.user_profiles` にレコードが挿入される。

**実装ファイル**: （Supabase SQL Editor で実行）

## 注意事項

- `service_role` キーはスーパー権限を持つため、フロントエンドには絶対に公開しないこと
- JWT Secret はバックエンドの JWT 検証に使用する。厳重に管理すること
- Apple OAuth の設定は Google OAuth より手順が複雑。Apple Developer アカウントが必要
- スキーマ適用は必ず開発用プロジェクトで実行してから本番へ適用すること
- RLS ポリシーが正しく設定されていない場合、他ユーザーのデータが見える可能性があるため、必ず確認すること
