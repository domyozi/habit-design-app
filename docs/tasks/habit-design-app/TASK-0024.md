## 実装詳細

### 1. Vercel フロントエンドデプロイ 🔵

**信頼性**: 🔵 *architecture.md デプロイメント設計 より*

#### 1-1. Vercel CLI セットアップ

```bash
npm i -g vercel
vercel login
```

#### 1-2. GitHubリポジトリとの連携

Vercelダッシュボードから以下を設定する:

- 対象リポジトリを選択
- Framework Preset: `Vite`
- Root Directory: `frontend/`
- Build Command: `npm run build`
- Output Directory: `dist`

#### 1-3. Vercel 環境変数設定

Vercelダッシュボード → Settings → Environment Variables に以下を設定する:

| 変数名 | 値 | 対象環境 |
|---|---|---|
| `VITE_SUPABASE_URL` | SupabaseプロジェクトURL | Production, Preview |
| `VITE_SUPABASE_ANON_KEY` | Supabase anon key（公開キー） | Production, Preview |
| `VITE_API_BASE_URL` | Railway の FastAPI URL | Production |

> **注意**: `VITE_` プレフィックスの変数はビルド時にバンドルされクライアントに露出する。機密情報（service role key、Claude APIキー等）は絶対に設定しないこと（NFR-101）。

#### 1-4. HTTPS確認（NFR-103）

Vercelはデプロイ時にHTTPS証明書を自動発行する。`http://` でアクセスした場合に `https://` へリダイレクトされることを確認する。

#### 1-5. Supabase Auth リダイレクトURL設定

Supabaseダッシュボード → Authentication → URL Configuration で以下を設定する:

- **Site URL**: `https://{vercel-domain}/`
- **Redirect URLs**: `https://{vercel-domain}/` を追加

---

### 2. Railway バックエンドデプロイ 🔵

**信頼性**: 🔵 *architecture.md デプロイメント設計・NFR-101 より*

#### 2-1. Railway CLI セットアップ

```bash
npm i -g @railway/cli
railway login
```

#### 2-2. プロジェクト設定

Railwayダッシュボードから以下を設定する:

- 対象リポジトリを選択
- Root Directory: `backend/`
- Start Command: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`

#### 2-3. Railway 環境変数設定

Railwayダッシュボード → Variables に以下を設定する:

| 変数名 | 値 | 機密度 |
|---|---|---|
| `SUPABASE_URL` | SupabaseプロジェクトURL | 低 |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key | **機密** |
| `SUPABASE_JWT_SECRET` | JWT検証シークレット | **機密** |
| `ANTHROPIC_API_KEY` | Claude API キー（NFR-101） | **機密** |
| `RESEND_API_KEY` | Resend APIキー | **機密** |
| `FRONTEND_URL` | Vercel デプロイURL（CORS許可） | 低 |

> **重要（NFR-101）**: `ANTHROPIC_API_KEY` はRailwayサーバー側のみで使用する。フロントエンドの環境変数には絶対に含めないこと。

#### 2-4. HTTPS確認（NFR-103）

Railwayはデプロイ時にHTTPS証明書を自動発行する。`https://{railway-domain}/docs` でSwagger UIにアクセスできることを確認する。

---

### 3. Supabase 本番設定更新 🔵

**信頼性**: 🔵 *architecture.md デプロイメント設計 より*

Supabaseダッシュボードで以下を更新する:

#### 3-1. Auth 設定

- **Site URL**: Vercel デプロイURLに更新
- **Redirect URLs**: Vercel デプロイURLを追加

#### 3-2. CORS設定

APIの許可オリジン（バックエンドの `CORS_ORIGINS` 設定）に Railway ドメインおよび Vercel ドメインを追加する。`backend/app/main.py` の `CORSMiddleware` 設定で `FRONTEND_URL` 環境変数を参照していることを確認する。

---

## 注意事項

- Vercel の Preview デプロイ（プルリクエスト用）には `VITE_API_BASE_URL` を本番RailwayのURLとは別に設定することを検討する
- Railway の `$PORT` 環境変数は Railway が自動付与するため、起動コマンドで必ず参照すること
- Supabase の `service_role` キーはRLSをバイパスする強力な権限を持つため、Railway環境変数以外には保存しないこと
- Google OAuthのコールバックURLをGoogle Cloud ConsoleでもVercelドメインに更新すること
