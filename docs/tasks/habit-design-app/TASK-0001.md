## 実装詳細

### 1. Vite プロジェクト作成 🔵

**信頼性**: 🔵 *architecture.md フロントエンドコンポーネント構成より*

プロジェクトルートで以下を実行してフロントエンドプロジェクトを作成する。

```bash
npm create vite@latest frontend -- --template react-ts
cd frontend
npm install
```

**実装ファイル**: `frontend/`

---

### 2. 必要パッケージのインストール 🔵

**信頼性**: 🔵 *architecture.md フロントエンドコンポーネント構成より*

```bash
# Tailwind CSS v3
npm install -D tailwindcss@3 postcss autoprefixer @tailwindcss/forms
npx tailwindcss init -p

# ルーティング
npm install react-router-dom@6

# データフェッチ
npm install @tanstack/react-query@5

# 状態管理
npm install zustand@4

# フォーム
npm install react-hook-form@7

# Supabase
npm install @supabase/supabase-js@2

# HTTP クライアント
npm install axios

# 日付処理
npm install date-fns

# アイコン
npm install lucide-react
```

**実装ファイル**: `frontend/package.json`

---

### 3. TypeScript 設定 🔵

**信頼性**: 🔵 *architecture.md フロントエンドコンポーネント構成より*

`frontend/tsconfig.json` を以下のように設定する。

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "useDefineForClassFields": true,
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "skipLibCheck": true,
    "moduleResolution": "bundler",
    "allowImportingTsExtensions": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "jsx": "react-jsx",
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noFallthroughCasesInSwitch": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  },
  "include": ["src"],
  "references": [{ "path": "./tsconfig.node.json" }]
}
```

**実装ファイル**: `frontend/tsconfig.json`

---

### 4. Vite 設定（パスエイリアス） 🔵

**信頼性**: 🔵 *architecture.md フロントエンドコンポーネント構成より*

`frontend/vite.config.ts` を以下のように設定する。

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path'

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
})
```

`path` モジュールの型定義のため `@types/node` も追加する。

```bash
npm install -D @types/node
```

**実装ファイル**: `frontend/vite.config.ts`

---

### 5. Tailwind CSS 設定 🔵

**信頼性**: 🔵 *architecture.md フロントエンドコンポーネント構成より*

`frontend/tailwind.config.js` を以下のように設定する。

```javascript
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [
    require('@tailwindcss/forms'),
  ],
}
```

`frontend/src/index.css` の先頭に Tailwind ディレクティブを追加する。

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

**実装ファイル**: `frontend/tailwind.config.js`, `frontend/src/index.css`

---

## 注意事項

- `@/` エイリアスは `tsconfig.json` と `vite.config.ts` の両方に設定すること
- Tailwind CSS は v3 を使用すること（v4 は互換性未確認）
- `.env.example` はコミット対象。実際の値を含む `.env` はコミットしないこと
- `npm run dev` 起動後、`http://localhost:5173` でアクセスできることを確認すること
