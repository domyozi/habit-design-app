## テストフロー詳細

### フロー1: 初回ユーザー体験（ストーリー1.1+1.2） 🔵

**信頼性**: 🔵 *user-stories 1.1/1.2 より*

1. Googleアカウントでログイン
2. オンボーディング画面でWanna Be入力（「毎朝6時に起きて英語とランニングを続けられている自分」）
3. 「AIに相談する」クリック → SSEでAI分析がストリーミング表示される
4. 目標候補3件が表示される → 「承認」で保存
5. ダッシュボードに遷移して習慣リストが表示される

**期待結果**: ログインから習慣リスト表示まで一連の操作が完了する

---

### フロー2: 毎日の習慣チェック（ストーリー2.1+3.1+3.2） 🔵

**信頼性**: 🔵 *user-stories 2.1/3.1/3.2 より*

1. ダッシュボードで今日のルーティンを確認
2. 習慣をチェック（✅）→ ストリーク更新が即時反映される
3. 習慣を未達成（❌）→ 未達成理由入力欄が表示される → 「残業で時間がなかった」入力して保存
4. 7日連続達成の習慣をチェック → バッジ獲得通知が表示される

**期待結果**: チェック操作が2秒以内に完了し（NFR-001）、バッジが付与される

---

### フロー3: 音声入力（ストーリー2.2） 🔵

**信頼性**: 🔵 *user-stories 2.2 より*

1. 音声入力ボタンをタップ（Chrome）
2. 「英語30分達成、筋トレはできなかった」と話す（またはテキスト入力）
3. AIが分類し対応する習慣ログが更新される
4. EDGE-003確認: 判断不能な入力をした場合に手動選択UIが表示される

**期待結果**: 音声/テキスト入力から習慣ログ更新が完了する

---

### フロー4: 週次レビュー（ストーリー5.1） 🔵

**信頼性**: 🔵 *user-stories 5.1 より*

1. 週次レビュー画面を開く
2. 「週次レビューを開始する」をクリック
3. AIフィードバックがSSEでストリーミング表示される
4. AI提案（時間帯変更）の「承認」をクリック → 習慣のscheduled_timeが更新される

**期待結果**: AIフィードバックが30秒以内に完了するか、ストリーミングで逐次表示される（NFR-002）

---

### フロー5: エラー時の動作確認（EDGE-001） 🔵

**信頼性**: 🔵 *EDGE-001 より*

1. Railway環境変数の `ANTHROPIC_API_KEY` を無効な値に変更（一時的）
2. Wanna Be分析を実行 → 「AI機能が現在利用できません。テキストのみ保存されました」が表示される
3. ダッシュボードでのチェックリスト操作が引き続き使用できることを確認
4. `ANTHROPIC_API_KEY` を元の値に戻す

**期待結果**: AI障害時でもトラッキング機能は継続動作する

---

### セキュリティ確認（NFR-101/102） 🔵

**信頼性**: 🔵 *NFR-101/102 より*

- ブラウザのDevToolsでネットワークタブを確認: `ANTHROPIC_API_KEY` が含まれないこと
- ユーザーAでログインし、ユーザーBのデータURLに直接アクセス → 404または403が返ること
