## タスク概要

Supabase Auth を使用した Google OAuth ログイン機能をフロントエンドに実装し、バックエンドに JWT 検証ミドルウェアを実装する。フロントエンドは Zustand で認証状態を管理し、ルーティングと連動させる。バックエンドは Supabase JWT を検証して認証済みユーザーの user_id を取得する依存関数を提供する。

#### 3. JWT 検証関数（security.py） 🔵

**信頼性**: 🔵 *NFR-101/102 JWT検証設計より*

`backend/app/core/security.py` を以下の内容で作成する。

```python
from datetime import datetime
from typing import Optional
from fastapi import Depends, HTTPException, status
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
from jose import JWTError, jwt
from app.core.config import settings

security = HTTPBearer()


def verify_token(token: str) -> Optional[str]:
    """
    Supabase JWT を検証し、user_id を返す。
    検証失敗時は None を返す。
    """
    try:
        payload = jwt.decode(
            token,
            settings.supabase_jwt_secret,
            algorithms=["HS256"],
            audience="authenticated",
        )
        user_id: str = payload.get("sub")
        if user_id is None:
            return None
        # 有効期限の確認
        exp = payload.get("exp")
        if exp is None or datetime.utcfromtimestamp(exp) < datetime.utcnow():
            return None
        return user_id
    except JWTError:
        return None


async def get_current_user(
    credentials: HTTPAuthorizationCredentials = Depends(security),
) -> str:
    """
    FastAPI 依存関数。
    Authorization ヘッダーの JWT を検証し、user_id を返す。
    未認証・無効トークンの場合は 401 UNAUTHORIZED を返す。
    """
    user_id = verify_token(credentials.credentials)
    if user_id is None:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Invalid or expired token",
            headers={"WWW-Authenticate": "Bearer"},
        )
    return user_id
```

**実装ファイル**: `backend/app/core/security.py`

#### 5. 認証状態管理（authStore.ts） 🔵

**信頼性**: 🔵 *architecture.md フロントエンドコンポーネント構成より*

`frontend/src/store/authStore.ts` を以下の内容で作成する。

```typescript
import { create } from 'zustand'
import { Session, User } from '@supabase/supabase-js'
import { supabase } from '@/lib/supabase'

interface AuthState {
  session: Session | null
  user: User | null
  isLoading: boolean
  signIn: () => Promise<void>
  signOut: () => Promise<void>
  initialize: () => void
}

export const useAuthStore = create<AuthState>((set) => ({
  session: null,
  user: null,
  isLoading: true,

  signIn: async () => {
    await supabase.auth.signInWithOAuth({
      provider: 'google',
      options: {
        redirectTo: `${window.location.origin}/auth/callback`,
      },
    })
  },

  signOut: async () => {
    await supabase.auth.signOut()
    set({ session: null, user: null })
  },

  initialize: () => {
    // 初回セッション取得
    supabase.auth.getSession().then(({ data: { session } }) => {
      set({ session, user: session?.user ?? null, isLoading: false })
    })

    // セッション変更の監視
    supabase.auth.onAuthStateChange((_event, session) => {
      set({ session, user: session?.user ?? null, isLoading: false })
    })
  },
}))
```

**実装ファイル**: `frontend/src/store/authStore.ts`

#### 7. アプリルーティング（App.tsx） 🔵

**信頼性**: 🔵 *architecture.md フロントエンドコンポーネント構成より*

`frontend/src/App.tsx` を以下の内容で作成する。

```tsx
import { useEffect } from 'react'
import { BrowserRouter, Routes, Route, Navigate } from 'react-router-dom'
import { useAuthStore } from '@/store/authStore'
import { Login } from '@/pages/Login'

// ダッシュボードは後続タスクで実装
const Dashboard = () => (
  <div className="p-8">
    <h1 className="text-2xl font-bold">ダッシュボード</h1>
    <p>実装予定</p>
  </div>
)

// オンボーディングは後続タスクで実装
const Onboarding = () => (
  <div className="p-8">
    <h1 className="text-2xl font-bold">オンボーディング</h1>
    <p>実装予定</p>
  </div>
)

// 認証コールバック処理
const AuthCallback = () => {
  useEffect(() => {
    // Supabase が URL フラグメントのトークンを自動処理する
    window.location.href = '/'
  }, [])
  return <div>認証処理中...</div>
}

// 認証済みユーザー専用ルート
const PrivateRoute = ({ children }: { children: React.ReactNode }) => {
  const { session, isLoading } = useAuthStore()
  if (isLoading) return <div>読み込み中...</div>
  if (!session) return <Navigate to="/login" replace />
  return <>{children}</>
}

export const App = () => {
  const initialize = useAuthStore((state) => state.initialize)

  useEffect(() => {
    initialize()
  }, [initialize])

  return (
    <BrowserRouter>
      <Routes>
        <Route path="/login" element={<Login />} />
        <Route path="/auth/callback" element={<AuthCallback />} />
        <Route
          path="/onboarding"
          element={
            <PrivateRoute>
              <Onboarding />
            </PrivateRoute>
          }
        />
        <Route
          path="/*"
          element={
            <PrivateRoute>
              <Dashboard />
            </PrivateRoute>
          }
        />
      </Routes>
    </BrowserRouter>
  )
}

export default App
```

**実装ファイル**: `frontend/src/App.tsx`

import pytest
from datetime import datetime, timedelta
from jose import jwt
from fastapi.testclient import TestClient
from app.main import app

TEST_JWT_SECRET = "test-secret-key-for-unit-tests"
TEST_USER_ID = "00000000-0000-0000-0000-000000000001"


def create_test_token(
    user_id: str = TEST_USER_ID,
    secret: str = TEST_JWT_SECRET,
    expires_delta: timedelta = timedelta(hours=1),
    audience: str = "authenticated",
) -> str:

        "exp": expire,
        "aud": audience,

## 注意事項

- `jose` ライブラリのバージョンは `python-jose[cryptography]==3.3.*` を使用すること
- Supabase JWT の audience は `"authenticated"` であることに注意
- テスト用の JWT Secret は本番の `SUPABASE_JWT_SECRET` と異なる値を使用すること
- `get_current_user` は `FastAPI Depends` 経由で使用し、直接呼び出さないこと
- フロントエンドの認証コールバック URL は Supabase の「Redirect URLs」に登録すること（`Authentication > URL Configuration`）
- Wanna Be 未設定のユーザーをオンボーディングへリダイレクトするロジックは後続タスクで実装
