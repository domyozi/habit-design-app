## タスク概要

ログイン画面を実装し、Google OAuth・Apple OAuthによる認証フローとオンボーディング遷移を構築する。ログイン成功後、Wanna Be未設定ユーザーは `/onboarding` へ、設定済みユーザーは `/` へ遷移する。モバイルファーストのレスポンシブデザインで実装する。

## 実装詳細

### 1. ログイン画面（Login.tsx）🔵

**信頼性**: 🔵 *REQ-101/102・user-stories 1.1 より*

Supabase OAuthを使用したログイン画面を実装する。

- 「Googleでログイン」ボタン: `supabase.auth.signInWithOAuth({ provider: 'google' })` を呼び出す
- 「Appleでログイン」ボタン（REQ-102）: `supabase.auth.signInWithOAuth({ provider: 'apple' })` を呼び出す
- OAuth完了後のコールバック処理: `supabase.auth.onAuthStateChange` でセッション変更を検知
- 遷移判定:
  - `GET /wanna-be` が404または空データ → `/onboarding` へ
  - データあり → `/` へ
- ローディング状態管理: OAuthリダイレクト中・API呼び出し中は `isLoading: true`

**実装ファイル**: `frontend/src/pages/Login.tsx`

---

### 2. ローディングスピナーコンポーネント（Spinner.tsx）🔵

**信頼性**: 🔵 *NFR-201 より*

汎用スピナーコンポーネントを実装する。

- Tailwind CSSの `animate-spin` を使用した円形スピナー
- `size` prop（`sm` / `md` / `lg`）でサイズ変更可能
- `color` prop（デフォルト: `white`）で色変更可能
- ボタン内インライン表示・全画面オーバーレイ表示の両方に対応

**実装ファイル**: `frontend/src/components/ui/Spinner.tsx`

---

### 3. モバイルファーストデザイン 🔵

**信頼性**: 🔵 *NFR-201 より*

- モバイル: 全画面表示（`min-h-screen flex flex-col items-center justify-center`）
- ボタン: 幅いっぱい（`w-full`）・高さ最低44px（NFR-202）
- アプリロゴ・キャッチコピー（「未来の自分から逆算して習慣を設計する」）を中央表示
- Tailwindのレスポンシブクラスでタブレット・PCでも適切に表示

## UI/UX要件

### ローディング状態 🔵

**信頼性**: 🔵 *NFR-201 より*

- OAuthリダイレクト中・遷移先判定中はボタンを `disabled` にしてスピナーをインライン表示する
- ボタンのテキストを「ログイン中...」に変更する

### エラー表示 🔵

- エラー発生時は画面上部または入力エリア下に赤色のエラーバナーを表示
- メッセージ: 「ログインに失敗しました。もう一度お試しください。」
- エラーバナーは5秒後に自動消去

### モバイル対応 🔵

**信頼性**: 🔵 *NFR-201/202 より*

- 全画面縦中央配置（`min-h-screen`）
- ボタン高さ最低44px（NFR-202）
- Safe Area対応（`pb-safe`等のiOS対応クラス）
