### 2. Wanna Be 分析SSEエンドポイント 🔵

**信頼性**: 🔵 *REQ-201/203・api-endpoints.md より*

**実装ファイル**: `backend/app/api/routes/wanna_be.py`（SSEエンドポイント追加）

- `POST /wanna-be/analyze`:
  - Wanna Beを `wanna_be` テーブルにUPSERT（version+1, is_current=true）
  - `StreamingResponse` でSSEを返す（Content-Type: text/event-stream）
  - Claude API障害時: テキスト保存のみ行い503を返す（EDGE-001）

---

### 3. 週次レビューSSEエンドポイント 🔵

**信頼性**: 🔵 *REQ-601/702・api-endpoints.md より*

**実装ファイル**: `backend/app/api/routes/ai_coach.py`

- `GET /ai/weekly-review/stream`:
  - week_startパラメータ（省略時は今週の月曜日）
  - weekly_reviewsテーブルにpending状態でINSERT
  - `StreamingResponse` でSSEを返す
  - done時にweekly_reviewsをcompleted/achievement_rate更新
  - Claude API障害時: `data: {"type":"error","error":"AI_UNAVAILABLE"}\n\n`

---

### 4. AIデータ制限の実装（REQ-605） 🔵

**信頼性**: 🔵 *REQ-605・NFR-101・design-interview.md より*

Claude APIに送信するデータは以下に限定:
- 習慣タイトルと達成/未達成の日付パターン（集計値）
- ストリーク日数
- 未達成理由テキスト（ユーザーが記入したもの）

送信禁止データ:
- メールアドレス・表示名等の個人情報
- Supabase user_id

## 注意事項

- Claude APIのストリーミングには `AsyncAnthropic` クライアントを使用すること
- FastAPIの `StreamingResponse` に `media_type="text/event-stream"` を指定すること
- SSEフォーマット: 各チャンクは `data: {JSON}\n\n` 形式
- テストでは `anthropic` モジュールをモック化してAPIコストを回避すること
