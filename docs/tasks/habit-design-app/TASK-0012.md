## タスク概要

フロントエンドの共通基盤として、FastAPI呼び出し用APIクライアント・Zustand認証状態ストア・React Router v6ルーティング・TanStack Query Providerを整備する。すべての認証済みAPIリクエストにSupabase JWTを自動付与し、未認証ユーザーをログイン画面へリダイレクトする仕組みを構築する。

## 実装詳細

### 1. APIクライアント（api.ts）🔵

**信頼性**: 🔵 *architecture.md・REQ-103 より*

axiosインスタンスを設定し、リクエスト／レスポンスインターセプターでJWT付与と401ハンドリングを行う。

- `baseURL`: 環境変数 `VITE_API_URL` から取得
- リクエストインターセプター: `supabase.auth.getSession()` でJWTを取得し `Authorization: Bearer {token}` を付与
- レスポンスインターセプター: 401レスポンス時に `supabase.auth.signOut()` を呼び出してログアウト
- ラッパー関数: `apiGet<T>`, `apiPost<T>`, `apiPatch<T>`, `apiDelete<T>`
- SSEストリーミング用: `createSSEStream(url: string, token: string): EventSource`

**実装ファイル**: `frontend/src/lib/api.ts`

---

### 2. Zustand認証状態ストア（authStore.ts）🔵

**信頼性**: 🔵 *TASK-0004・REQ-103 より*

TASK-0004で作成済みのストアを拡張し、以下の状態を管理する。

- `session`: Supabaseセッションオブジェクト
- `user`: Supabaseユーザー情報
- `profile`: DBのusersテーブルのプロフィール情報
- `isLoading`: 認証確認中フラグ
- `isAuthenticated`: 認証済みフラグ（sessionが存在するかどうか）
- `setSession(session)`: セッション設定アクション
- `signOut()`: ログアウトアクション（SupabaseのsignOut呼び出し）
- `initialize()`: アプリ起動時に既存セッションを復元

**実装ファイル**: `frontend/src/store/authStore.ts`

---

### 3. React Routerルーティング（App.tsx）🔵

**信頼性**: 🔵 *architecture.md より*

React Router v6の `createBrowserRouter` または `BrowserRouter` を使用してルーティングを設定する。

| パス | コンポーネント | 認証要否 |
|---|---|---|
| `/login` | Login.tsx | 不要 |
| `/onboarding` | Onboarding.tsx | 不要 |
| `/` | Dashboard.tsx | 要認証 |
| `/wanna-be` | WannaBe.tsx | 要認証 |
| `/weekly-review` | WeeklyReview.tsx | 要認証 |
| `/settings` | Settings.tsx | 要認証 |

`PrivateRoute` コンポーネント: `isAuthenticated` が false かつ `isLoading` が false の場合、`<Navigate to="/login" replace />` でリダイレクト。ローディング中はスピナー表示。

**実装ファイル**: `frontend/src/App.tsx`

---

### 4. アプリエントリーポイント（main.tsx）🔵

**信頼性**: 🔵 *architecture.md より*

`QueryClient` インスタンスを作成し、`QueryClientProvider` でアプリ全体をラップする。`staleTime` は5分に設定。

**実装ファイル**: `frontend/src/main.tsx`

## 単体テスト要件

### テストケース1: JWTヘッダー付与の確認 🔵

**信頼性**: 🔵 *REQ-103・NFR-101 より*

**Given**: Supabaseセッションが存在し、`access_token` が `"test-jwt-token"` である
**When**: `apiGet<T>("/habits")` を呼び出す
**Then**: リクエストヘッダーに `Authorization: Bearer test-jwt-token` が付与されていること

---

### テストケース2: セッションなし時のヘッダー未付与 🔵

**信頼性**: 🔵 *REQ-103 より*

**Given**: Supabaseセッションが存在しない（null）
**When**: `apiGet<T>("/habits")` を呼び出す
**Then**: `Authorization` ヘッダーが付与されないこと

---

### テストケース3: 401レスポンス時のサインアウト 🔵

**信頼性**: 🔵 *NFR-101 より*

**Given**: APIがステータス401を返す
**When**: `apiGet<T>("/habits")` を呼び出す
**Then**: `supabase.auth.signOut()` が呼び出されること

---

### テストケース4: PrivateRouteの未認証リダイレクト 🔵

**信頼性**: 🔵 *NFR-101・architecture.md より*

**Given**: `isAuthenticated` が false・`isLoading` が false
**When**: `PrivateRoute` でラップされた `/` にアクセスする
**Then**: `/login` へリダイレクトされること

---

### テストケース5: PrivateRouteの認証済みアクセス 🔵

**信頼性**: 🔵 *NFR-101 より*

**Given**: `isAuthenticated` が true
**When**: `PrivateRoute` でラップされた `/` にアクセスする
**Then**: ダッシュボードコンポーネントが表示されること
