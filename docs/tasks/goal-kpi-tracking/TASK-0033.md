## 実装詳細

### 1. voice_classifier.py KPI 候補マッチング 🔵

**信頼性**: 🔵 *REQ-LOG-003・EDGE-KPI-006 より*

**実装ファイル**: `backend/app/services/voice_classifier.py`

既存の `kpi_update` 分類後に、発話から抽出した `unit_hint` を使って
`kpis` テーブルの `unit` カラムと照合し、候補リストを返すロジックを追加する。

```python
async def match_kpi_candidates(
    unit_hint: str,
    user_id: str,
    supabase: Client
) -> list[dict]:
    """unit_hint に一致する KPI 候補を返す"""
    if not unit_hint:
        return []
    
    kpis = supabase.table("kpis").select("id, title, unit").eq("user_id", user_id).eq("is_active", True).execute()
    
    candidates = []
    for kpi in kpis.data:
        kpi_unit = (kpi.get("unit") or "").lower()
        hint = unit_hint.lower()
        # 単位の部分一致（例: "kg" ⊆ "kg" または "キロ" ⊆ "kg"の読み仮名対応）
        if hint in kpi_unit or kpi_unit in hint:
            candidates.append({
                "kpi_id": kpi["id"],
                "title": kpi["title"],
                "unit": kpi.get("unit"),
            })
    
    return candidates
```

### 2. ai_service.py 週次レビュープロンプト拡張 🔵

**信頼性**: 🔵 *REQ-REVIEW-001〜003・NFR-KPI-102 より*

**実装ファイル**: `backend/app/services/ai_service.py`

週次レビュー生成時に呼び出されるプロンプト構築関数に KGI/KPI コンテキストを追加する。
個人情報（タイトル）は送信せず、達成率・週次平均・習慣との相関のみを送信する。

```python
async def build_weekly_review_prompt_with_kgi(
    user_id: str,
    habit_stats: dict,
    supabase: Client
) -> str:
    """KGI進捗を含む週次レビュープロンプトを構築する"""
    # KGI 一覧と達成率を取得
    goals = supabase.table("goals").select("metric_type, target_value, current_value, target_date").eq("user_id", user_id).not_.is_("target_date", "null").execute()
    
    kgi_stats = []
    for g in goals.data:
        achievement_rate = None
        if g.get("target_value") and g.get("current_value"):
            achievement_rate = min(100, (g["current_value"] / g["target_value"]) * 100)
        kgi_stats.append({
            "metric_type": g["metric_type"],
            "achievement_rate": achievement_rate,
        })
    
    # KPI 週次平均を取得
    from datetime import date, timedelta
    week_start = str(date.today() - timedelta(days=7))
    kpis = supabase.table("kpis").select("id, metric_type, target_value, tracking_frequency").eq("user_id", user_id).eq("is_active", True).execute()
    
    kpi_stats = []
    for kpi in kpis.data:
        logs = supabase.table("kpi_logs").select("value").eq("kpi_id", kpi["id"]).gte("log_date", week_start).execute()
        values = [l["value"] for l in logs.data]
        from statistics import mean
        kpi_stats.append({
            "metric_type": kpi["metric_type"],
            "target_value": kpi.get("target_value"),
            "weekly_avg": round(mean(values), 2) if values else None,
            "tracking_frequency": kpi["tracking_frequency"],
        })
    
    # プロンプト構築（タイトルなし）
    kgi_section = f"KGI進捗（{len(kgi_stats)}件）: {kgi_stats}"
    kpi_section = f"KPI週次状況: {kpi_stats}"
    
    return f"""
{kgi_section}
{kpi_section}
習慣達成状況: {habit_stats}

上記のデータを分析して:
1. KGIへの今週の貢献度と来週の重点をコメントしてください
2. KPIの達成要因（習慣との相関）を分析してください
3. 習慣の調整提案を具体的に生成してください（change_time/add_habit/remove_habit の範囲で）
"""
```
