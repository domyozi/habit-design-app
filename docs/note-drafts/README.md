# Note 記事ドラフト

## 方針

タスク完了ごとに記事を1本書く。
「プロのアーキテクトがこう考えてこう作った」という視点で、
判断の背景・トレードオフ・ハマりどころを残す。

## ファイル命名規則

```
{TASK-ID}-{slug}.md
```

例: `TASK-0004-supabase-jwt-fastapi.md`

## 記事の構成テンプレート

1. **タイトル** — 検索されやすい具体的な技術名を入れる
2. **この記事でわかること** — 3行以内
3. **背景・設計の意図** — なぜこの構成を選んだか
4. **実装のポイント** — コード付きで解説
5. **ハマったこと・解決法** — 再現性のある失敗談
6. **アーキテクト視点のまとめ** — 次に活かせる判断軸
