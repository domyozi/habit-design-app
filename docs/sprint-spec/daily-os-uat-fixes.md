# Sprint Spec: Daily OS — UAT Fix Sprints

## 1. プロダクト概要

**一言説明**: UATで発見された不具合・UX問題を優先度順に修正し、Daily OS をすべてのユーザーが支障なく使える状態にする。
**対象ユーザー**: Daily OS の既存ユーザーおよび新規ユーザー（モバイル含む）
**コアバリュー**: アプリの基本操作が PC・モバイル双方で阻害なく動作し、ユーザーが習慣設計の本質的なループ（計画→実行→振り返り）を完結できること
**成功の定義**: P0 ブロッカーが解消され、P1 の主要 UX 問題が修正され、新規ユーザーが初日から迷わず使えること

## 3. スプリント計画

### Sprint 1: P0 ブロッカー修正

**目標**: モバイルユーザーが Mandala を操作でき、Evening Tab で複数行テキストを入力でき、Settings でどのデバイスからも習慣を編集できる状態にする。

#### タスク
- [ ] F-01: Mandala グリッドモバイル対応 — 390px 幅のスマートフォンで 9x9 グリッドの全セルが表示・タップできる
- [ ] F-02: Evening Tab テキストエリア修正 — Gap・Insight・Tomorrow の各フィールドで改行を含む複数行テキストが入力できる
- [ ] F-03: Settings 編集ボタンのタッチ対応 — タッチデバイスで TodoManager の編集(✎)・非表示(—)ボタンが常時表示され操作できる

#### 完了基準（Evaluator が使う）
- ブラウザを 390px 幅にリサイズしたとき、Mandala タブのグリッドが横スクロールまたはスケールされて全セルが表示される
- Mandala グリッドの任意のセルをクリック/タップしたとき、セルの状態が変化する
- Evening Tab の Gap フィールドに改行（Shift+Enter または textarea の自然な改行）が入力でき、複数行が保持される
- Evening Tab の Insight フィールド・Tomorrow フィールドでも同様に複数行入力できる
- Settings ページ（TodoManager）を 390px 幅で表示したとき、各習慣行の編集ボタンと非表示ボタンが常時見える（ホバーなしで表示される）
- タッチイベントで編集ボタンを押したとき、習慣の編集モードが起動する

---

### Sprint 2: P1 主要 UX 修正

**目標**: 習慣チェック・目標設定・月次分析・AI 機能の各フローが正確かつ安全に動作し、ユーザーが意図しないデータ損失を起こさない状態にする。

#### タスク
- [x] F-04: Morning Tab 表示順修正 — 習慣チェックリストがジャーナル（テキストエリア）より上に表示される
- [x] F-05: Home 画面 Primary Target 設定 — FocusCard に「目標を設定する」ボタンが表示され、クリックで設定フォームが開く
- [x] F-06: Monthly 分析の全習慣対応 — isMust フラグに関わらず全習慣が月次集計グラフに含まれる
- [x] F-07: WannaBeAnalysis データソース修正 — 月次 AI 分析が Supabase から取得した WannaBe データを使用する
- [x] F-08: AI Replace 確認ダイアログ — 「AI で習慣を置換」実行前に「既存の習慣がすべて置き換えられます。続けますか？」確認ダイアログが表示される
- [x] F-09: 体重目標ユーザー設定 — Settings または Profile 画面で体重目標値を入力・保存でき、Home 画面に反映される
- [x] F-10: Morning Tab レポート生成の遷移告知 — ボタン押下前または押下後に「Monthly タブに移動します」という通知またはモーダルが表示される

#### 完了基準（Evaluator が使う）
- Morning Tab を開いたとき、ページ上部に習慣チェックリストが、下部にジャーナルエリアが表示される
- Home 画面の FocusCard にプレースホルダーではなく操作可能なボタンまたはリンクがあり、クリックで目標入力フォームが開く
- Monthly タブで非 Must 習慣（Evening 習慣など）の達成率データが表示される
- WannaBe タブで登録した目標が Monthly タブの AI 分析に反映される（新規ユーザーでも空ではなくデータが表示される）
- AI 習慣置換ボタンを押したとき、確認ダイアログが表示され、キャンセルすると習慣が変更されない
- Settings に体重目標入力フィールドがあり、値を変更して保存すると Home 画面の体重表示が更新される
- Morning Tab の「レポートを生成」ボタンを押したとき、Monthly タブへの移動前に通知が表示される

### Sprint 3: P2 ポリッシュ + localStorage 移行

**目標**: 細部の品質を高め、データが複数デバイス間で同期されるようにし、新規ユーザーの初回体験を改善する。

- [x] F-12: Evening Tab プログレスバー追加 — Evening Tab の各セクションに記入率を示すプログレスバーが表示される
- [x] F-13: ログイン画面バリュープロポジション — ログインフォームの上部または横にアプリの説明・主要機能紹介が表示される
- [x] F-14: 「今日をリセット」確認ダイアログ — リセットボタン押下時に確認ダイアログが表示され、キャンセルするとリセットされない
- [x] F-15: HabitSelectSheet 再表示対応 — Mandala 画面でスキップ後も再度「習慣を選択」シートを開くボタンまたは手順が存在する
- [x] F-16: iOS Shortcuts JWT トークン表示 UI — Settings または Profile 画面でアクセストークンを表示・コピーできる UI が追加される
- [ ] F-17: granularity 設定の API 移行 — /api/user-context に granularity フィールドなしのため localStorage 維持（TODO コメント追加済み）
- [ ] F-18: Mandala デイリーチェックの API 移行 — バックエンドの daily-check エンドポイント未実装のためスキップ

#### 完了基準（Evaluator が使う）
- Home 画面のコーチメッセージが当日の習慣チェック率に応じて少なくとも 3 パターン（低/中/高）で変化する

- granularity 設定を変更・ページリロード後も設定値が保持される（localStorage をクリアしても API から復元される）
- Mandala で習慣をチェックしてページリロードしても、チェック状態が /api/mandala から復元される
- Mandala の tracked 状態がリロード後も維持される

## 4. ユーザーストーリー

### F-01: Mandala グリッドモバイル対応

**As** スマートフォンユーザー
**I want** 390px 幅の画面でも Mandala グリッドを読んで操作したい
**So that** 外出先でも習慣の進捗を確認・チェックできる

**受け入れ条件**:
- [ ] 390px 幅でグリッドの全 81 セルが表示される（横スクロールまたは縮小）
- [ ] セルのテキストが読める（文字が重ならない・切れない）
- [ ] セルをタップしたとき状態が変化する

### F-02: Evening Tab テキストエリア修正

**As** 夜の振り返りをするユーザー
**I want** Gap・Insight・Tomorrow フィールドに複数行のテキストを入力したい
**So that** 思ったことを自由に書き出せる

**受け入れ条件**:
- [ ] Gap フィールドで改行を含むテキストが入力できる
- [ ] Insight・Tomorrow フィールドも同様に複数行入力できる
- [ ] 入力内容が保存されて再訪時に復元される

### F-03: Settings 編集ボタンのタッチ対応

**As** スマートフォンから設定を変更するユーザー
**I want** 習慣リストの編集・非表示ボタンをタッチで操作したい
**So that** ホバーが効かないデバイスでも習慣を管理できる

**受け入れ条件**:
- [ ] 390px 幅でボタンが常時表示される（opacity-0 でない）
- [ ] ボタンをタップして編集モードが起動する
- [ ] ボタンをタップして非表示が切り替わる

### F-04: Morning Tab 表示順修正

**As** 朝に習慣チェックをするユーザー
**I want** ページを開いたらすぐにチェックリストが見えるようにしたい
**So that** ジャーナルをスクロールせずに主要アクションに集中できる

**受け入れ条件**:
- [ ] Morning Tab を開いたとき、チェックリストが最上部に表示される
- [ ] ジャーナルエリアはチェックリストの下に配置される
- [ ] 既存の機能（チェック・ジャーナル保存）が変わらず動作する

### F-05: Home 画面から Primary Target を設定

**As** 新規ユーザーまたは目標を変更したいユーザー
**I want** Home 画面から直接 Primary Target を設定したい
**So that** Morning Tab を探さずに今日の目標を決められる

**受け入れ条件**:
- [ ] FocusCard に目標未設定時は設定を促すボタンまたはテキストが表示される
- [ ] そのボタンから目標入力フォームが開く
- [ ] 設定後 FocusCard に目標が表示される

### F-06: Monthly 分析の全習慣対応

**As** 全習慣の達成状況を把握したいユーザー
**I want** Must 以外の習慣も月次分析に含めてほしい
**So that** Evening・非 Must タスクの改善もデータで把握できる

**受け入れ条件**:
- [ ] Monthly タブで Evening 習慣の達成データが表示される
- [ ] isMust: false の Morning 習慣も集計対象になる
- [ ] 既存の Must 習慣データが正しく表示され続ける

### F-07: WannaBeAnalysis データソース修正

**As** WannaBe（なりたい自分）を登録して月次振り返りをするユーザー
**I want** AI 月次分析が私の最新 WannaBe データを読んでほしい
**So that** 意味のある分析メッセージが表示される

**受け入れ条件**:
- [ ] WannaBe タブで登録した目標テキストが Monthly AI 分析に使われる
- [ ] 新規ユーザーが WannaBe を登録後、Monthly 分析が空にならない
- [ ] localStorage の wannabe:goals キーに依存しない

### F-08: AI Replace 確認ダイアログ

**As** AI 提案を試したいユーザー
**I want** 全習慣が置換される前に確認を求めてほしい
**So that** 誤操作で既存の習慣を失わない

**受け入れ条件**:
- [ ] AI 置換ボタン押下時に確認ダイアログが表示される
- [ ] 「キャンセル」を選ぶと習慣データが変更されない
- [ ] 「確認」を選ぶと通常通り置換が実行される

### F-09: 体重目標のユーザー設定化

**As** 体重管理をしているユーザー
**I want** 自分の体重目標値を設定したい
**So that** 他のユーザーのデータではなく自分の目標値が表示される

**受け入れ条件**:
- [ ] Settings または Profile 画面に体重目標の入力フィールドがある
- [ ] 値を保存すると Home 画面の体重目標表示が更新される
- [ ] 初期値は空またはオンボーディング時に設定を促す

### F-10: Morning Tab レポート生成の遷移告知

**As** Morning Tab を使っているユーザー
**I want** レポート生成ボタンが別タブへ移動させることを事前に知りたい
**So that** 意図せず作業中の Morning Tab を離れない

**受け入れ条件**:
- [ ] 「レポートを生成」ボタン押下時に「Monthly タブに移動します」という通知またはモーダルが表示される

**As** Daily OS を毎日使うユーザー

**受け入れ条件**:
- [ ] mandala:tracked の読み書きが /api/mandala 経由になる
- [ ] localStorage クリア後もリロードで tracked 状態が復元される

| ナビゲーション | 全主要画面（Home/Morning/Evening/Monthly/Settings）へのアクセス | 100% |

## 6. スコープ外（明示的に作らないもの）

- バックエンドのアーキテクチャ変更（API エンドポイントの追加・変更は最小限）
